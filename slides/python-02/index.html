What is OOP?

Object-Oriented Programming

OOP is a way of structuring your code around objects — self-contained units that combine data and the functions that operate on that data. Instead of thinking "what steps do I run?", you think "what things does my program work with, and what can they do?"

  • 🧩 A way to organize code around things (objects) instead of loose functions
  • 📦 Each object bundles its own data (what it knows) and behavior (what it can do)
  • 🔁 The most widely used programming paradigm — Python, Java, C++, JavaScript all support it
  • 🔧 Built on 4 core principles — we'll look at each one next

"OOP doesn't replace functions. It gives them a home."

What is a Class?

A class is the tool you use to do OOP. It's a template (blueprint) for creating objects. You define the template once, then create as many objects from it as you want — each with its own data.

📦 Encapsulation

Bundle data and behavior together. Expose what's needed, hide the rest. Think about driving a car — you interact with a simple interface (steering wheel, pedals) while the engine, transmission, and wiring stay hidden under the hood.

✅ What you see (interface)

🎯 Steering wheel
🦶 Pedals (gas, brake)
📊 Speedometer
🔑 Ignition

🔒 What's hidden (internals)

⚙️ Engine
🔗 Transmission
⚡ Wiring
💧 Fuel injection
  • • You drive the car without knowing how the engine works
  • • A class works the same way — users call methods, internals stay hidden
  • • Convention: prefix with _underscore to mark something as internal
  • • This protects your data from being accidentally changed by code outside the class

🌳 Inheritance

Build new classes from existing ones. The child inherits everything the parent has — all its attributes and methods — and can add or change things on top. This avoids duplicating code when multiple classes share common behavior.

🐾 Animal
name · speak()
🐕 Dog
+ fetch()
🐈 Cat
+ purr()
  • • Dog IS AN Animal — it inherits name and speak() automatically
  • • Dog adds its own method: fetch(). Cat adds purr().
  • • You write shared logic once in the parent. Children reuse it.
  • • Rule: use inheritance when the child truly is a type of the parent

"Don't rewrite. Extend."

🎭 Polymorphism

Same method name. Different behavior depending on the object.

The word comes from Greek: poly (many) + morph (forms). When a parent class defines a method like speak(), each child class can override it with its own version. You call the same method on different objects, and Python automatically runs the right version.

speak()
↙ ↓ ↘
🐕
Dog
"Woof!"
🐈
Cat
"Meow!"
🦆
Duck
"Quack!"
  • • You call animal.speak() — Python picks the right version based on the object's type
  • • This happens automatically when you override a parent's method in a child class
  • • You don't "apply" polymorphism — it's a natural result of inheritance + method overriding
  • • This is powerful because your code can work with any Animal without knowing the specific type

Anatomy of a Class

Every class has the same three building blocks: attributes (data), methods (actions), and special methods (Python hooks). The class is the template — when you create an object from it, that object is called an instance.

s = Student("Ali", 20) → s is an instance of the Student class.

Student

📋 Attributes (data it holds)

name, age, score

Each object gets its own copy of these

⚙️ Methods (things it can do)

calculate_average() · is_passing() · display()

Functions that live inside the class and operate on its data

🔧 Special Methods

__init__ → runs when you create an object (setup)

__repr__ → controls how the object displays itself

self → refers to "this specific object"

Every class follows this structure: data + methods + special methods.

Understanding __init__ and self

These are the two concepts that trip up beginners most. __init__ is the setup method — it runs automatically when you create an object, setting up its initial data. self is how the object refers to itself — "my name", "my age", not someone else's.

class Student: 1 Defines the template
def __init__(self, name, age): 2 Runs automatically when object is created
self.name = name 3 Save data onto this object
self.age = age
def greet(self): 4 A method — function on this object
return f"Hi, I'm {self.name}" 5 self.name = this object's name
s = Student("Ali", 20)   # __init__ runs → self = s
s.greet()              # → "Hi, I'm Ali"

__init__ = the setup method. It runs once, automatically, the moment you create an object.
self = "me, this specific object." Every method gets it as the first argument automatically — you never pass it yourself.

Bonus Concepts (quick flags)

You'll see these in the code notebook. Just know they exist for now — you don't need to memorize them yet.

Python lets you hook your classes into built-in operations (like len(), print(), and indexing with []) by defining special "dunder" methods. And decorators let you add behavior to functions or methods without changing their code.

🔌

__dunder__ methods

Special methods that plug your class into Python's built-in operations

Define __len__ → len(obj) works.
Define __getitem__ → obj[0] works.
Define __str__ → print(obj) shows something useful.
🏷️

@property

Makes a method look like a simple attribute — no parentheses needed

student.full_name instead of
student.full_name() — cleaner to read.
🎁

@decorator

Wraps a function with extra behavior without changing the function itself

Gift wrapping — the gift stays the same, the wrapper adds something

These are tools you'll use after you're comfortable writing basic classes. We'll practice them in the coding notebook.
