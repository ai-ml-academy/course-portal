  • • Mutable — you can change, add, or remove items after creation
  • • Any type — a list can hold ints, strings, even other lists

  • • ⚠️ Python uses indentation (4 spaces) to know what's inside each block

Loops

Loops are like an assembly line — each item comes through, gets the same treatment, and moves on.

for loop

"Do this for each item"

names = ["Ali", "Sara"]

for name in names:
    print(name)

# With range()
for i in range(5):
    print(i) # 0,1,2,3,4

while loop

"Keep going while condition is true"

count = 0

while count < 3:
    print(count)
    count += 1

# ⚠️ Don't forget to
# update the condition!

🔧 Power tools: enumerate() and zip()

# enumerate: get index + value
for i, name in enumerate(["age", "income"]):
    print(f"Column {i}: {name}")

# zip: loop over two lists at once
for name, score in zip(["Ali", "Sara"], [85, 92]):
    print(f"{name}: {score}")

Comprehensions

A shortcut for building lists and dicts from loops — same result, fewer lines.

📋 List Comprehension

❌ The long way

squares = []
for x in range(5):
    squares.append(x ** 2)
→

✅ Comprehension

squares = [x ** 2 for x in range(5)]

You can add a filter: [x for x in items if x > 0]

📖 Dict Comprehension

names = ["age", "income", "score"]

# Map each name to its length
name_lengths = {name: len(name) for name in names}
# → {"age": 3, "income": 6, "score": 5}
  • • List: [expression for item in iterable if condition]
  • • Dict: {key: value for item in iterable if condition}
  • • ⚠️ If it doesn't fit on one line or needs complex logic — use a regular loop instead

Lambda Functions

A lambda is a mini function in one line — no name, no def, just input → output. Think of it as a sticky note with one quick instruction.

Regular function

def double(x):
    return x * 2
→

Lambda version

double = lambda x: x * 2

Where lambdas shine: inside other functions

# Sort a list of tuples by the second element
students = [("Ali", 85), ("Sara", 92), ("Ravi", 78)]

sorted(students, key=lambda s: s[1])
# → [("Ravi", 78), ("Ali", 85), ("Sara", 92)]

# Filter with lambda
list(filter(lambda x: x > 80, [78, 85, 92]))
# → [85, 92]

# Transform with lambda
list(map(lambda x: x * 2, [1, 2, 3]))
# → [2, 4, 6]

"If your lambda needs more than one line, use a regular def function instead."

Functions

A function is like a recipe — you define it once with a name, ingredients (inputs), and steps, then anyone can use it anytime with their own ingredients.

def calculate_average(scores): 1 Name + parameters
"""Return the average of a list of scores.""" 2 Docstring (optional docs)
total = sum(scores) 3 Body — the work
return total / len(scores) 4 Output — send a value back
# Calling the function
result = calculate_average([85, 92, 78])
print(result)  # → 85.0
  • • def — keyword that starts a function definition
  • • Parameters — placeholders in the definition: scores
  • • Arguments — actual values you pass in: [85, 92, 78]
  • • return — sends a result back to the caller. No return = returns None

Functions — Going Deeper

📥 Default Arguments

def greet(name, greeting="Hello"):
    return f"{greeting}, {name}!"

greet("Ali")              # → "Hello, Ali!"
greet("Ali", "Welcome")   # → "Welcome, Ali!"

🔍 Type Hints (documentation, not enforced)

def calculate_average(scores: list[float]) -> float:
    return sum(scores) / len(scores)

📦 Scope — where variables live

🌍 Global scope

name = "Ali"  # visible everywhere

🏠 Local scope (inside function)

x = 10  # only exists here

Rules

  • • Variables inside a function are local — they disappear when the function ends
  • • A function can read global variables
  • • A function cannot change global variables (without global keyword)
  • • Best practice: pass data in via parameters, send data out via return

Functions — *args & **kwargs

Sometimes you don't know how many arguments a function will receive. Python has two special prefixes for this.

*args — Variable Positional Args

Collects extra positional arguments into a tuple.
Think: "I'll take any number of items."

def total(*args):
    print(args)  # a tuple
    return sum(args)

total(10, 20)       # → 30
total(1, 2, 3, 4)  # → 10

**kwargs — Variable Keyword Args

Collects extra keyword arguments into a dict.
Think: "I'll take any named settings."

def build_profile(**kwargs):
    print(kwargs) # a dict

build_profile(
    name="Ali",
    age=25,
    role="analyst"
)
# → {"name": "Ali", "age": 25,
#    "role": "analyst"}

🔧 Using both together

def log_event(event, *args, **kwargs):
    print(f"Event: {event}")
    print(f"Extra args: {args}")    # tuple
    print(f"Options: {kwargs}")    # dict

log_event("login", "admin", 42, ip="10.0.0.1", secure=True)

File I/O

Almost every project starts by reading data and ends by writing results. Think of it like opening a notebook — you open it, read or write, and close it when done.

📄

data.txt

→ open()
🐍

Python

→ write()
📄

output.txt

✅ The safe way: with statement (auto-closes the file)

📖 Reading

with open("data.txt", "r") as f:
    content = f.read()

# Read line by line
with open("data.txt") as f:
    for line in f:
        print(line.strip())

✏️ Writing

with open("out.txt", "w") as f:
    f.write("Hello!\n")

# Append (don't overwrite)
with open("out.txt", "a") as f:
    f.write("New line\n")
