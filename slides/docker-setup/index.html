Install on macOS

Official guide docs.docker.com/desktop/setup/install/mac-install/
  1. Download Docker Desktop — pick Apple silicon (M1–M4) or Intel from the guide above.
  2. Open the .dmg and drag Docker.app to /Applications.
  3. Start it once (macOS will prompt for your admin password to install the privileged helper).
  4. Verify from Terminal once the daemon is ready.
Terminal — steps 3 and 4# start the daemon (background app; no window to keep open)
open -a Docker

# verify
docker --version
docker compose version
docker run --rm hello-world

Install on Windows 10 / 11

Official guide docs.docker.com/desktop/setup/install/windows-install/

Two parts: enable WSL 2, then install Docker Desktop with the WSL 2 backend. Run all Docker commands from inside Ubuntu, not PowerShell. 

1 · PowerShell (Admin) — enable WSL 2wsl --install
# reboot when prompted; Ubuntu first-launch asks for a username + password

wsl --status            # "Default Version: 2"
wsl --list --verbose    # Ubuntu should show VERSION 2
2 · Run the installer + start the daemon# run the wizard from the official guide above
# leave "Use WSL 2 instead of Hyper-V" CHECKED — only checkbox that matters; reboot if prompted

& "C:\Program Files\Docker\Docker\Docker Desktop.exe" -Autostart
wsl -d Ubuntu           # drop into your Ubuntu distro
3 · Ubuntu (WSL) — verifydocker --version
docker compose version
docker run --rm hello-world
Use the Ubuntu shell, not PowerShell. Keep your project files inside WSL (~/projects/...) — editing from C:\Users\you\... works but is much slower.

Install on Linux

Official guides Ubuntu · Debian · Fedora

Native containers — no VM, no Desktop. Install Docker Engine, then add yourself to the docker group so sudo isn't on every command.

Ubuntu / Debian

Docker Engine via apt

# remove any old/distro Docker packages
sudo apt-get remove docker docker-engine docker.io containerd runc 2>/dev/null

# set up Docker's official apt repo
sudo apt-get update
sudo apt-get install -y ca-certificates curl gnupg
sudo install -m 0755 -d /etc/apt/keyrings
curl -fsSL https://download.docker.com/linux/ubuntu/gpg | \
    sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg
sudo chmod a+r /etc/apt/keyrings/docker.gpg

echo "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] https://download.docker.com/linux/ubuntu $(. /etc/os-release && echo \"$VERSION_CODENAME\") stable" \
  | sudo tee /etc/apt/sources.list.d/docker.list > /dev/null

# install
sudo apt-get update
sudo apt-get install -y docker-ce docker-ce-cli containerd.io \
    docker-buildx-plugin docker-compose-plugin

# run docker without sudo
sudo usermod -aG docker $USER
newgrp docker            # picks up the new group in THIS shell
Fedora / RHEL / Rocky / Alma

Docker Engine via dnf

sudo dnf -y install dnf-plugins-core
sudo dnf config-manager --add-repo \
    https://download.docker.com/linux/fedora/docker-ce.repo

sudo dnf install -y docker-ce docker-ce-cli containerd.io \
    docker-buildx-plugin docker-compose-plugin

sudo systemctl enable --now docker
sudo usermod -aG docker $USER
newgrp docker

Verify your install

Three commands. If all three are green, your install is done — no matter which OS you came from. Run them in a fresh terminal. 
Terminal · any OS# 1 — CLI is installed and on PATH
docker --version
# → Docker version 24.0.0 or newer

# 2 — Compose v2 plugin is loaded
docker compose version
# → Docker Compose version v2.20+ (note: TWO words, no hyphen)

# 3 — the daemon can pull, run, and clean up a container
docker run --rm hello-world
expected — hello-world tailHello from Docker!
This message shows that your installation appears to be working correctly.

Bonus — one real-image smoke test

docker run --rm python:3.11-slim python -c "print(1 + 1)"
# → 2  · confirms the daemon can pull, run code, and clean up

Common gotchas

The six errors you're most likely to hit on install day. If your verify step failed, scan this list first.
Cannot connect to the Docker daemon
Daemon isn't running. Mac: open -a Docker · Linux: sudo systemctl start docker.
permission denied on /var/run/docker.sock
Linux — join the docker group: sudo usermod -aG docker $USER && newgrp docker.
docker-compose: command not found
Use docker compose (two words). The hyphenated v1 binary is deprecated.
Hyper-V / VT-x error on Windows
Virtualisation disabled in BIOS. Reboot into firmware, enable VT-x / AMD-V / SVM Mode.
Docker is eating my disk
docker system df shows usage. Clean with docker system prune.
port is already allocated
Something else holds the port — find & kill, or remap with -p 8001:8000. Next slide.

Free a port that's "already in use"

Three moves: find what holds the port, kill it (or remap), prefer docker stop when a container is the culprit.
macOS · Linux 
# find PID on 8000
lsof -iTCP:8000 -sTCP:LISTEN

# kill in one line
lsof -ti :8000 | xargs kill -9
Windows · PowerShell 
# find PID on 8000 (last column)
netstat -ano | findstr :8000

# kill in one line
Get-NetTCPConnection -LocalPort 8000 | % { Stop-Process -Id $_.OwningProcess -Force }
If a container holds it 
docker ps --filter "publish=8000"
docker stop <container-name>

# or just remap the host port
docker run -p 8001:8000 ...
Don't kill what you don't recognise. If the process isn't yours, remap the host side (-p 8001:8000) instead of killing.

Post-install cheatsheet

The day-to-day commands, grouped. Daemon & system in teal; images in amber; containers in green; Compose in navy.

Daemon & system

docker --version
installed CLI version
docker compose version
confirm Compose v2 plugin
docker info
full daemon status & config
docker system df
disk usage by category
docker system prune
nuke unused everything
open -a Docker
start daemon on macOS
sudo systemctl restart docker
restart daemon on Linux

Images

docker images
list local images
docker pull <image>
download from a registry
docker build -t <name> .
build from a Dockerfile
docker tag <src> <dst>
add another tag to an image
docker push <image>
upload to a registry
docker rmi <image>
delete a local image
docker image prune
remove dangling images (safe)

Containers

docker ps
list running containers
docker ps -a
list all (incl. stopped)
docker run <image>
start a new container
docker exec -it <c> bash
shell into a running container
docker logs -f <c>
follow container stdout/stderr
docker stop <c>
SIGTERM then SIGKILL
docker rm <c>
delete a stopped container
docker container prune
remove all stopped (safe)

Compose

docker compose up
start the whole stack
docker compose up --build
rebuild images first
docker compose ps
services + their status
docker compose logs -f <svc>
follow one service's logs
docker compose exec <svc> sh
shell into a running service
docker compose down
stop & remove (keep volumes)
docker compose down -v
also drop named volumes
