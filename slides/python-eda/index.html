What is EDA?

Exploratory Data Analysis is the process of summarizing and visualizing a dataset to understand its structure, identify patterns, and detect anomalies — before building any model.

Key questions EDA answers:

  • • What are the distributions of individual features?
  • • Where are the missing values and outliers?
  • • Which features correlate with the target?
  • • Are features correlated with each other (multicollinearity)?
  • • Is the target balanced (for classification)?

Skipping EDA is the most common source of poor ML results — you end up modeling data you don't understand.

Distribution Shapes

The shape of a histogram or KDE determines which transforms and models are appropriate.

Symmetric (Normal)

Mean ≈ Median. Examples: heights, measurement errors.

Right-skewed (Positive)

Mean > Median. Examples: income, house prices. Often fixed with log transform.

Left-skewed (Negative)

Mean < Median. Examples: age at retirement, scores on easy exam.

Bimodal

Two subpopulations mixed. Mean falls where little data exists. Consider splitting groups.

Box Plot & Violin Plot

Box Plot — five-number summary + outlier detection

$$\text{Outlier if:}\; x < Q_1 - 1.5 \cdot \text{IQR} \;\;\text{or}\;\; x > Q_3 + 1.5 \cdot \text{IQR}$$

  • • Median off-center in the box → skewed distribution
  • • Grouped: sns.boxplot(data=df, x='value', y='category', orient='h')

Violin Plot — density shape + quartiles

  • • A violin plot is a mirrored KDE + box plot combined. Width = density of data at that value.
  • • Wide = many observations. Narrow = few. Dashed lines = quartiles.
  • • Group B above is bimodal (two bulges) — a box plot would hide this completely.
  • • Code: sns.violinplot(data=df, x='value', y='group', orient='h', inner='quartile')

Correlation — Pearson vs Spearman

Pearson ($r$)

Measures linear association only.

$$r = \frac{\sum(x_i - \bar{x})(y_i - \bar{y})}{\sqrt{\sum(x_i - \bar{x})^2 \cdot \sum(y_i - \bar{y})^2}}$$

Assumes continuous data, roughly normal. Sensitive to outliers.

Spearman ($\rho$)

Measures monotonic association (consistently increasing/decreasing, even if curved).

Computed on ranks, not raw values. Robust to outliers. Works on ordinal data.

A log curve (always increasing) → Spearman ≈ 1, but Pearson < 1.

df.corr(method='pearson')    # default — linear
df.corr(method='spearman')   # monotonic — robust to nonlinearity

# With p-values
from scipy.stats import pearsonr, spearmanr
r, p = pearsonr(df['x'], df['y'])

Correlation Patterns — Pearson vs Spearman

Pearson $r$ only captures linear relationships. Spearman $\rho$ captures monotonic ones. Both miss non-monotonic patterns. Each subplot shows computed $r$ and $\rho$ values.

Row 1: Linear patterns — Pearson and Spearman agree.
Row 2: Non-linear patterns. Green = Spearman captures the monotonic curve that Pearson underestimates. Rose = non-monotonic — both measures fail. Always plot your data.

The EDA Workflow

A structured approach — move from individual features to relationships to the full picture.

  1. Overview: df.shape, df.info(), df.describe(), df.head()
  2. Univariate: Histogram + boxplot for each numeric feature. value_counts() for categoricals. Check distributions, outliers, balance.
  3. Bivariate: Scatter plots of features vs target. Boxplots of target grouped by categorical features. Which features appear predictive?
  4. Multivariate: Correlation heatmap. pairplot if ≤10 features. Identify multicollinearity.
  5. Findings: Document missing patterns, outliers, key correlations, data quality issues.

Identifying Outliers

Outliers are observations that deviate significantly from the rest. They can be legitimate extremes or data errors.

$$\text{IQR} = Q_3 - Q_1 \qquad \text{Outlier if:}\; x < Q_1 - 1.5 \cdot \text{IQR} \;\;\text{or}\;\; x > Q_3 + 1.5 \cdot \text{IQR}$$

Q1 = df['price'].quantile(0.25)
Q3 = df['price'].quantile(0.75)
IQR = Q3 - Q1
outliers = df[(df['price'] < Q1 - 1.5*IQR) | (df['price'] > Q3 + 1.5*IQR)]
  • • Box plots show outliers directly — points beyond whiskers
  • • Always investigate before removing — outlier in house prices may be a legitimate mansion
  • • Options: remove, cap (winsorize), log-transform, or leave — depends on downstream model

Missing Values

Understanding how much and why determines your strategy.

df.isnull().sum().sort_values(ascending=False)    # counts
(df.isnull().sum() / len(df) * 100)             # percentages
sns.heatmap(df.isnull(), cbar=False, yticklabels=False) # pattern
Missing %Strategy
< 5%Drop rows — minimal impact
5–30%Impute: median (skewed), mean (normal), mode (categorical)
> 30%Drop column or create binary missing indicator

MCAR (Missing Completely at Random) — safe to impute. MNAR (Missing Not at Random, e.g., high earners skip income) — requires domain-specific handling.

Class Imbalance

When one class dominates, the model learns to ignore the minority class entirely because predicting the majority is the easiest way to minimize loss.

Example: fraud detection (95% Not Fraud, 5% Fraud)

What happensWhy
Model predicts "Not Fraud" for everythingDuring training, the model sees 19 non-fraud examples for every 1 fraud. It learns that always predicting "Not Fraud" minimizes overall error.
95% accuracy — looks great on paperAccuracy counts all correct predictions equally. Getting the 9,500 non-fraud cases right overwhelms the 500 missed frauds.
0% recall on fraud — useless in practiceThe model never learned the fraud pattern. It has no ability to detect what you actually care about.
Decision boundary is biased toward majorityThe loss gradient is dominated by the majority class. The model's decision boundary shifts to classify almost everything as the majority.

How to detect and respond

# Detect
df['target'].value_counts(normalize=True)
sns.countplot(data=df, x='target')
  • • Metrics: use precision, recall, F1, ROC-AUC — not accuracy
  • • Stratified split: train_test_split(stratify=y) preserves class ratios in both sets
  • • Class weights: class_weight='balanced' penalizes misclassifying the minority more heavily
  • • Resampling: oversample minority (SMOTE) or undersample majority

matplotlib — Figure, Axes & Core Plots

matplotlib organizes every plot as Figure (canvas) + Axes (plot area). All customization happens through ax.

import matplotlib.pyplot as plt

fig, ax = plt.subplots(figsize=(8, 5))
ax.plot(x, y, label='revenue')
ax.set_title('Monthly Revenue')
ax.set_xlabel('Month')
ax.set_ylabel('Revenue ($)')
ax.legend()
ax.grid(True, alpha=0.3)
plt.tight_layout()
plt.savefig('plot.png', dpi=150)

Complete EDA Script Template

import pandas as pd
import matplotlib.pyplot as plt
import seaborn as sns

df = pd.read_csv('data.csv')
target = 'target_col'
num_cols = df.select_dtypes('number').columns.drop(target, errors='ignore')
cat_cols = df.select_dtypes('object').columns

# 1. Overview
print(df.shape)
print(df.info())
print(df.describe())
print(df.isnull().sum().sort_values(ascending=False))

# 2. Univariate — numeric distributions
for col in num_cols:
    fig, axes = plt.subplots(1, 2, figsize=(10, 4))
    sns.histplot(df[col], kde=True, ax=axes[0])
    sns.boxplot(x=df[col], ax=axes[1])
    plt.suptitle(col); plt.tight_layout(); plt.show()

# 3. Univariate — categorical frequencies
for col in cat_cols:
    print(df[col].value_counts())
    sns.countplot(data=df, y=col, order=df[col].value_counts().index)
    plt.show()

# 4. Bivariate — numeric features vs target
for col in num_cols:
    sns.scatterplot(data=df, x=col, y=target, alpha=0.5)
    plt.show()

# 5. Bivariate — categorical features vs target
for col in cat_cols:
    sns.boxplot(data=df, x=col, y=target)
    plt.show()

# 6. Multivariate — correlation heatmap
plt.figure(figsize=(10, 8))
sns.heatmap(df[num_cols].corr(), annot=True, cmap='coolwarm', center=0, fmt='.2f')

# 7. Missing value pattern
sns.heatmap(df.isnull(), cbar=False, yticklabels=False)

# 8. Target balance (classification)
print(df[target].value_counts(normalize=True))
sns.countplot(data=df, x=target)
