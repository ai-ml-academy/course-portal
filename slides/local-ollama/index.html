Local LLMs with Ollama

Pull a model, chat in the terminal, call it from Python — all on your laptop

What is a local LLM

A local LLM runs on your machine. The model weights sit in ~/.ollama/models, inference uses your CPU or GPU, and the only thing that leaves your laptop is whatever you choose to share. No API key. No per-token bill. Works on a plane.

THE HONEST TRADEOFF
A 7B local model won't out-reason GPT-5 on tricky tasks — frontier cloud models are larger, smarter, and pricier for a reason. But for learning, prototyping, and most production RAG, a local 3B–8B is plenty. Build local first; reach for cloud only when quality genuinely requires it.

Tools for running local LLMs

Three popular ways to run open-weight models on a laptop. They wrap the same underlying inference engines (llama.cpp, gguf) but differ in interface. This deck uses Ollama — simplest CLI, OpenAI-compatible API out of the box.
OLLAMA

CLI + daemon

One-command install. Background server on localhost:11434 with an OpenAI-compatible HTTP API. Best default if you write code.

LM STUDIO

Desktop GUI

Point-and-click app for browsing, downloading, and chatting with models. Also exposes a local OpenAI-compatible server. Best if you want zero CLI.

HUGGING FACE

Library + hub

Lowest level: transformers in Python, manual loading and inference. Best when you need full control of tokeniser, sampling, or quantisation.

How Ollama works

Ollama is a small background daemon. The CLI and your Python code both talk to it over HTTP on localhost:11434. The daemon loads model weights from disk, runs inference, and streams results back — all on your machine.
Terminal ollama run …
Python openai SDK · requests
HTTP POST → :11434 ↓
Ollama daemon always running · auto-starts on boot
load weights into RAM ↓
Model files ~/.ollama/models · 1–8 GB each

The daemon starts automatically after install (background app on macOS / Windows, systemd service on Linux). You never start it manually — you just type ollama run … or call the API and the daemon answers.

Install Ollama

Two options to install Ollama:
  • Grab the GUI installer at ollama.com/download and follow the on-screen steps.
  • Run the one-line command for your OS in a terminal (see below).
macOS
Run in Terminal. Daemon auto-starts on login.
curl -fsSL https://ollama.com/install.sh | sh
Linux
Run in your shell. Installs a systemd service called ollama and starts it.
curl -fsSL https://ollama.com/install.sh | sh
Windows
Run in PowerShell. Installs a background service that starts with Windows.
irm https://ollama.com/install.ps1 | iex
Verify the daemon# should return JSON; empty list of models on first run
curl http://localhost:11434/api/tags

# or open in a browser — should say "Ollama is running"
open http://localhost:11434/

Find a model

Browse the catalog at ollama.com/library. Pick by parameter count and free RAM — a rough rule is ~1 GB of RAM per 1 B parameters for Q4-quantised models.
Model nameSize on diskRAM neededGood for
llama3.2:3b~2 GB~4 GB freegeneral chat · tight laptops
llama3.2:1b~1.3 GB~2 GB freefastest · weakest reasoning
qwen2.5:7b~4.7 GB~8 GB freeinstruction-following · code
mistral:7b~4.4 GB~8 GB freegeneral purpose · long-time community favourite
nomic-embed-text~270 MB~1 GB freeembeddings (768-dim) for RAG

How to read the name: family:tag. llama3.2:3b = the Llama 3.2 family, the 3-billion-parameter variant. Omitting the tag picks the default (usually the most popular size). Many models also ship explicit quantisation tags like :q4_K_M or :fp16 if you want to control the precision.

Pull and manage models

Models are downloaded once and cached under ~/.ollama/models. The CLI gives you everything you need to add, list, and remove them. 
Manage your local models# download a model (~2 GB)
ollama pull llama3.2:3b

# also pull an embedding model
ollama pull nomic-embed-text

# see what you have
ollama list
# NAME                      ID            SIZE     MODIFIED
# llama3.2:3b               a80c4f17acd5  2.0 GB   2 minutes ago
# nomic-embed-text:latest   0a109f422b47  274 MB   1 minute ago

# see what's currently loaded in RAM
ollama ps

# pull updates for a model you already have
ollama pull llama3.2:3b

# remove a model to free disk space
ollama rm llama3.2:3b

Run a model from the terminal

ollama run opens an interactive chat. Append a quoted prompt for a one-shot answer. Type /bye to exit a chat session. 
Interactive chat$ ollama run llama3.2:3b
>>> What is retrieval-augmented generation?
RAG is a pattern where you fetch relevant text from your own
documents at query time and paste it into the LLM's prompt, so
the model answers from your data rather than its training data.

>>> /bye
$ # back at your shell
One-shot prompt$ ollama run llama3.2:3b "Explain RAG in one sentence."
RAG fetches relevant chunks from your documents and pastes them
into the prompt so the LLM answers from grounded context.

$ # piping works too
$ cat question.txt | ollama run llama3.2:3b

The first call after pulling a model takes a few extra seconds — the daemon loads the weights into RAM. Subsequent calls are fast until the model is evicted (Ollama unloads idle models automatically). ollama ps shows what's currently resident.

Use Ollama from a Python notebook

Two equivalent ways. The OpenAI SDK path is the course standard — your code works against Ollama locally and any cloud provider after a single .env change. 
Installpip install openai           # OpenAI-compatible path
pip install ollama           # or the native client
Approach 1 — OpenAI SDK (recommended)from openai import OpenAI

client = OpenAI(
    base_url="http://localhost:11434/v1",
    api_key="ollama",    # required but ignored
)

resp = client.chat.completions.create(
    model="llama3.2:3b",
    messages=[
        {"role": "system", "content": "You are concise."},
        {"role": "user",   "content": "What is a local LLM?"},
    ],
)
print(resp.choices[0].message.content)
Approach 2 — native ollama clientfrom ollama import Client

client = Client(host="http://localhost:11434")

resp = client.chat(
    model="llama3.2:3b",
    messages=[{"role": "user", "content": "What is a local LLM?"}],
)
print(resp.message.content)
Embeddings (either approach)vec = client.embeddings.create(
    model="nomic-embed-text",
    input="refund policy and cancellation rules",
).data[0].embedding

print(len(vec))    # → 768

Command cheatsheet

Everything you actually need, in one place.
ollama pull <model>

ollama list
show all locally downloaded models
ollama ps
show models currently loaded in RAM
ollama run <model>

verify the daemon is responding
tail -f ~/.ollama/logs/server.log
follow daemon logs (macOS)
sudo systemctl restart ollama
restart the daemon (Linux)
COMMON GOTCHAS
Port is 11434 (not 11343). API errors if the model isn't pulled — run ollama list first. Cold start adds a few seconds while the model loads into RAM.
