How AI, ML & Deep Learning Relate

These terms are nested, not synonyms. AI is the broadest category. Machine Learning is a subset that learns from data. Neural Networks are brain-inspired ML models. Deep Learning is neural networks with many layers — and it contains the architectures that power modern AI.

Model Families at a Glance

All ML models fall into two big groups: Classical ML (no neural networks) and Neural Networks (including deep learning). Each family has strengths for particular types of tasks.

Key Milestones in AI History

The field is 70 years old, but most of the breakthroughs you hear about happened in the last decade. Here's the timeline that explains how we got from simple math to ChatGPT and beyond.

Why did the Deep Learning era start in 2012?
  • AlexNet won ImageNet 2012 by a massive margin — it used a deep CNN trained on GPUs and crushed every traditional computer vision method, dropping the error rate from 26% to 16%. This proved deep networks could dramatically outperform hand-engineered features when given enough data and compute.
  • GPU availability made it practical — NVIDIA GPUs (originally for gaming) turned out to be perfect for the parallel matrix multiplications that neural networks need. Training that would take months on CPUs could now finish in days.
  • Large datasets (ImageNet) provided the fuel — ImageNet gave researchers 1.2 million labeled images across 1,000 categories. Previous datasets were too small for deep networks to show their advantage over classical methods.

Sequence & Multimodal Tasks

AI models handle two broad categories of input/output patterns. Sequence-to-sequence tasks stay within the same modality — text in, text out (translation, summarization, text generation). Multimodal (also called cross-modal) tasks translate between different modalities — text to images, images to text, and more. Here's how these patterns break down.

SEQ-TO-SEQ

Translation / Summarization

Input: Text (English)
Output: Text (French)
Example: "Hello" → "Bonjour"

ONE-TO-MANY

Text Generation

Input: Short prompt
Output: Long-form content
Example: "Write an essay about..." → 500 words

MANY-TO-ONE

Classification / Sentiment

Input: Full document
Output: Single label
Example: Review text → "Positive"

MULTIMODAL

Text → Image

Input: Text prompt
Output: Generated image
Example: "A cat on Mars" → 🖼️

MULTIMODAL

Image + Text → Text

Input: Photo + question
Output: Description/answer
Example: Photo + "What's this?" → "A golden retriever"

MULTIMODAL

Image → Image

Input: Source image
Output: Transformed image
Example: Photo → Style-transferred painting

Reinforcement Learning

In reinforcement learning, an agent learns by trial and error — taking actions in an environment and receiving rewards or penalties. It's how AlphaGo learned to beat world champions and how LLMs get aligned to human preferences.

Typical Uses

  • Game playing — AlphaGo, Atari, StarCraft
  • Robotics — learn to walk, grasp, navigate
  • Control systems — optimize HVAC, traffic lights
  • Autonomous driving — decision-making in traffic

Connection to LLMs

RLHF (Reinforcement Learning from Human Feedback) uses RL to align language models with human preferences — it's how ChatGPT became helpful instead of just completing text.

Human raters rank outputs → reward model trained → LLM optimized via PPO/DPO

Part 4

From RNNs to Transformers to LLMs

The architecture evolution that changed everything

Sequence Modeling: The RNN Era

Before Transformers, Recurrent Neural Networks (RNNs) were the go-to for sequential data like text, audio, and time series. They process one element at a time, passing a hidden state forward — like reading a book word by word and trying to remember everything.

LSTMs & GRUs: Partial Fixes

People were trying to model long sequences. LSTMs (proposed 1997, widely adopted ~2013) and GRUs (2014) introduced gating mechanisms that helped gradients flow through longer sequences, partially solving the vanishing/exploding gradient problem. LSTM was published in 1997 by Hochreiter & Schmidhuber, but it became widely practical around 2013-2015 when GPU compute made training on large datasets feasible. However, the fundamental sequential nature of RNNs remained — you still had to process tokens one at a time, which meant you couldn't parallelize training across GPUs. This sequential bottleneck was the real limitation that Transformers would ultimately solve.

The Transformer Revolution

In 2017, the paper "Attention Is All You Need" introduced the Transformer architecture. The key insight: instead of processing tokens one at a time, use an attention mechanism that lets every token look at every other token simultaneously.

RNN: Sequential

Transformer: Parallel (Self-Attention)

RNN Problems (Before 2017)

  • Sequential processing — must process token 1 before token 2 before token 3... no way to speed up with more GPUs
  • Vanishing gradients — by token 100+, the model has "forgotten" what it read at the beginning
  • Exploding gradients — training becomes unstable with very long sequences
  • Can't scale — adding more hardware doesn't help because each step depends on the previous one

Transformer Solutions (2017+)

  • Parallel processing — all tokens processed simultaneously, not one by one. Training is massively faster
  • Self-attention — every token can attend to every other token regardless of distance. Token 1 can directly influence token 1000
  • No vanishing gradient over long sequences — attention provides a direct path for gradients to flow through
  • Scales with compute — more GPUs = faster training. This is what enabled billion-parameter models
  • Led directly to LLMs — the ability to scale up training is what made GPT, Claude, and Llama possible
Why This Single Change Mattered So Much

Parallel attention means Transformers can be trained on thousands of GPUs simultaneously, enabling models with hundreds of billions of parameters. RNNs could never have reached this scale because their sequential nature meant adding GPUs didn't help. The Transformer architecture is the single innovation behind GPT, Claude, Llama, BERT, and every modern LLM.

How LLMs Actually Work

An LLM generates text one token at a time. It takes your input, predicts the most likely next token from its entire vocabulary (which can contain 50,000+ words/subwords), appends it to the context, and repeats. This process is called autoregression.

You provide input tokens (the "context")

2
Model outputs a probability for EVERY word in its vocabulary
The model scores all ~50,000 possible next tokens. Higher bar = higher probability.

floor — 18%
bed — 12%
roof 8%
...
3
Select the highest-probability token and append it to the context
The cat sat on the mat ? ← new token added
4
🔄 Feed the new context back into the model → repeat Steps 1-3 until a stop token is generated
Why "mat" and not "floor"?
During pretraining, the model read billions of sentences. In those sentences, "The cat sat on the" was followed by "mat" far more often than "floor." The probabilities above come from these learned patterns in the training data.
What is "attention"?
Think of reading a sentence and highlighting the important words. The Transformer does exactly this — for each prediction, it scores how relevant each previous word is. For "The cat sat on the ___", it pays more attention to "cat" and "sat" than to "the" or "on." This is what makes Transformers so powerful.
Context Window
Max tokens the model sees at once. Up to 1M+ in modern models.
Temperature
0 = always pick top token. 1 = sample randomly (more creative).
Vocabulary
~50,000-100,000 subword tokens. Every generation step scores ALL of them.

The LLM Training Pipeline

Building a useful LLM isn't one step — it's five. Each stage transforms the model from a raw text predictor into a helpful, safe assistant you can deploy to users.

  1. Pretraining

    The model reads trillions of tokens from the internet, books, and code, learning to predict the next word. This is where it acquires knowledge and language ability.

    Example: Input: "The cat sat on the" → Predicts: "mat"  |  Cost: $2M–$100M+  |  Time: Weeks to months on thousands of GPUs

  2. Instruction Fine-tuning

    Train on curated prompt/response pairs so the model follows instructions instead of just completing text. This turns a text completer into an assistant.

    Example: Prompt: "Summarize this article" → Response: [well-structured summary]  |  Data: 10K–1M examples

  3. RLHF / Alignment

    Human raters rank model outputs from best to worst. A reward model learns these preferences, then the LLM is optimized with RL (PPO or DPO) to produce outputs humans prefer.

    Goal: Make the model helpful, harmless, and honest  |  This is what prevents harmful outputs

  4. Task-specific Fine-tuning

    Adapt the aligned model to your specific domain or task using techniques like LoRA or QLoRA. This is the step you would do as a Gen AI engineer.

    Example: Fine-tune for medical Q&A, legal document analysis, or customer support  |  Cost: $10–$1,000

  5. Inference (Deployment)

    Deploy the model to serve real users via API or on-device. Optimize for latency, throughput, and cost. This is where the model does actual work.

    Tools: vLLM, TGI, Ollama, cloud APIs  |  Metrics: Tokens/second, time to first token, cost per 1M tokens

Terminology, tools, jobs & your roadmap

Open Source vs Closed Source

The performance gap between open-weight and closed-source models has been shrinking rapidly. By early 2026, top open models like DeepSeek-R1 and Llama 4 rival commercial APIs on many benchmarks. The choice depends on your constraints: cost, privacy, control, and regulatory requirements.

Key AI/ML Terminology

These are the terms you'll encounter constantly. They're grouped into four categories so you can build a structured vocabulary from day one.

Data Terms
  • Features — input variables the model uses to make predictions (e.g., age, income, pixel values)
  • Labels / Target — the correct answer you want the model to learn (e.g., "spam" or "$350K")
  • Dataset — a structured collection of examples, each with features and optionally labels
  • Batch — a small group of examples processed together in one training step
Model Terms
  • Parameters — learned numbers ("knobs") inside the model; more = bigger model (GPT-4: ~1.8T)
  • Weights — the specific parameter values learned during training
  • Architecture — the structure/design of the model (e.g., Transformer, CNN, Random Forest)
  • Layers — stacked processing stages; deeper = more abstract understanding
Training Terms
  • Loss — a number measuring how wrong the model's prediction is (lower = better)
  • Optimizer — the algorithm that decides how to adjust weights (e.g., Adam, SGD)
  • Epoch — one complete pass through the entire training dataset
  • Learning Rate — how big each weight update step is (too high = chaotic, too low = slow)
  • Backpropagation — algorithm that calculates which weights to adjust and by how much
Inference Terms
  • Tokens — subword units the model reads/writes (~1 token ≈ 0.75 English words)
  • Context Window — max tokens the model can see at once (Claude: 200K, Gemini: 1M+)
  • Temperature — controls randomness: 0 = deterministic, 1 = creative, >1 = chaotic
  • Prompt — the input you give to the model
  • Latency — time until you get a response (seconds)
  • Throughput — tokens generated per second

AI Job Roles in 2026

The AI field has distinct job families. They overlap, but each has a different core focus. Understanding where you're headed helps you prioritize what to learn.

Gen AI Engineer

50% Software Engineering + 50% LLM Work

  • Build AI-powered applications (APIs, chatbots, agents)
  • RAG pipelines, prompt engineering, evaluation
  • Fine-tuning, embedding strategies, tool orchestration
  • Docker, CI/CD, testing, deployment

You don't invent architectures — you USE them.

ML Engineer / Researcher

Pushes state-of-the-art

  • Design new model architectures
  • Train foundation models from scratch
  • Publish papers, run large-scale experiments
  • Deep math: linear algebra, calculus, statistics

Typically PhD-level, at research labs (OpenAI, DeepMind, FAIR)

Data Scientist

Insights from data

  • Exploratory data analysis (EDA)
  • Statistical modeling, A/B testing
  • Dashboards, business intelligence
  • Classical ML (trees, regression, clustering)

Focus on understanding data and driving business decisions

MLOps Engineer

Infrastructure & reliability

  • Model serving, scaling, monitoring
  • CI/CD for ML pipelines
  • Kubernetes, cloud infrastructure
  • Data pipelines, feature stores, cost optimization

Keeps models running reliably in production at scale
